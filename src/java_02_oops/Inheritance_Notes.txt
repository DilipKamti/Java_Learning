INHERITANCE IN JAVA (Interview Notes)
=================================

1) What is Inheritance?
----------------------
Inheritance means one class (child/subclass) acquires properties (fields) and behaviors (methods)
of another class (parent/superclass) using the keyword 'extends'.

Simple words:
- Parent gives common features
- Child reuses + can add its own features

Syntax:
class Child extends Parent { }

Benefits:
- Code reusability
- Less duplication
- Easy maintenance
- Supports Polymorphism (method overriding)


2) Types of Inheritance in Java
-------------------------------

A) Single Inheritance
Parent -> Child

class A { }
class B extends A { }

B) Multilevel Inheritance
A -> B -> C

class A { }
class B extends A { }
class C extends B { }

C) Hierarchical Inheritance
One parent, multiple children

class A { }
class B extends A { }
class C extends A { }

D) Multiple Inheritance (NOT supported using classes)
A + B -> C  (NOT allowed in Java with classes)

class C extends A, B { }  // ‚ùå Not allowed

E) Hybrid Inheritance (NOT supported using classes)
Combination of multiple types (needs multiple inheritance)
‚ùå Not supported with classes
‚úÖ Can be achieved using Interfaces


3) Diamond Problem (Very Important)
-----------------------------------
Diamond Problem happens when a class tries to inherit from two classes which both have the same method.

Example idea:
    A
   /   B   C
   \ /
    D

If both B and C have method show(), then D is confused:
Which show() should it call? B's or C's?

Because of this ambiguity:
‚úÖ Java does NOT allow multiple inheritance with classes.


4) How Java solves Multiple Inheritance?
----------------------------------------
Java supports multiple inheritance using Interfaces.

Example:
interface A { default void show() { } }
interface B { default void show() { } }

class C implements A, B {
   @Override
   public void show() {
      // must resolve conflict
      A.super.show();
      B.super.show();
   }
}


5) super keyword
----------------
super is used to access parent class things:
- super()           -> calls parent constructor
- super.methodName() -> calls parent method
- super.variable     -> accesses parent variable


6) Method Overriding (Runtime Polymorphism)
-------------------------------------------
Overriding means:
Child class provides its own implementation of a parent method.

Rules:
- Same method name
- Same parameters
- Same return type (or covariant)
- Must be in parent-child relationship

Access modifier rule:
Child method cannot be more restrictive.
(public -> cannot become protected/private)


7) Upcasting (Important)
------------------------
Parent reference can hold child object:

Parent p = new Child();

Runtime method call:
p.overriddenMethod();  // Child version runs

But:
p.childOnlyMethod();   // ‚ùå not allowed (reference type decides accessible methods)


8) final keyword in inheritance
-------------------------------
- final class -> cannot be extended
- final method -> cannot be overridden


9) Quick Interview One-Liners
-----------------------------
- Java is single inheritance with classes, multiple inheritance via interfaces.
- Inheritance supports code reuse and runtime polymorphism.
- Diamond problem is the reason multiple inheritance with classes is not allowed.
